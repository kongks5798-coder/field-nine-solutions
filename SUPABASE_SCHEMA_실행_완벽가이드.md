# 📝 Supabase 스키마 실행 완벽 가이드 (초보자용)

## 🎯 목표
Supabase SQL Editor에서 스키마 파일 2개를 실행하여 데이터베이스 테이블 만들기

---

## 🎬 1단계: SQL Editor 열기

### 1-1. Supabase 프로젝트 대시보드 확인
1. Supabase 대시보드에 접속되어 있는지 확인
2. **"field-nine-solutions"** 프로젝트가 선택되어 있는지 확인
3. 프로젝트 대시보드 화면이 보여야 합니다

### 1-2. 왼쪽 메뉴에서 SQL Editor 찾기
1. 화면 왼쪽에 세로로 나열된 메뉴가 있습니다
2. 여러 메뉴 아이콘과 텍스트가 보입니다:
   - Table Editor
   - **SQL Editor** ← 이걸 찾으세요!
   - Authentication
   - Storage
   - 등등...
3. **"SQL Editor"** 메뉴를 찾습니다
   - 아이콘: 데이터베이스 또는 SQL 관련 아이콘
   - 텍스트: **"SQL Editor"** 또는 **"SQL 편집기"**
4. **"SQL Editor"**를 클릭합니다

✅ **성공 확인:** SQL Editor 페이지가 열리면 성공입니다!

---

## 🎬 2단계: 새 쿼리 만들기

### 2-1. SQL Editor 화면 확인
SQL Editor 페이지가 열리면:
- 왼쪽: 쿼리 목록 또는 폴더 구조
- 가운데: SQL 코드를 입력하는 큰 텍스트 영역
- 오른쪽: 결과 표시 영역 (아직 비어있음)

### 2-2. "New query" 버튼 찾기
1. 화면 상단 또는 왼쪽 상단을 보세요
2. **"+ New query"** 또는 **"새 쿼리"** 버튼을 찾습니다
   - 버튼 색상: 보통 초록색 또는 파란색
   - 아이콘: **"+"** 또는 **"+"** 기호
   - 텍스트: **"New query"** 또는 **"새 쿼리"**
3. **"+ New query"** 버튼을 클릭합니다

✅ **성공 확인:** 새로운 빈 쿼리 창이 열리면 성공입니다!

---

## 🎬 3단계: 첫 번째 스키마 파일 열기

### 3-1. 프로젝트 폴더 열기
1. 컴퓨터에서 **파일 탐색기**를 엽니다
   - Windows 키 + E 키를 누르거나
   - 작업 표시줄에서 폴더 아이콘 클릭
2. 다음 경로로 이동합니다:
   ```
   C:\Users\polor\field-nine-solutions
   ```
   - 또는 프로젝트 폴더가 있는 위치

### 3-2. supabase 폴더 찾기
1. 프로젝트 폴더 안에서 **"supabase"** 폴더를 찾습니다
2. **"supabase"** 폴더를 더블클릭해서 엽니다

### 3-3. schema.sql 파일 찾기
1. supabase 폴더 안에서 **"schema.sql"** 파일을 찾습니다
2. 파일 이름: `schema.sql`
3. **"schema.sql"** 파일을 더블클릭해서 엽니다
   - 메모장, VS Code, 또는 기본 텍스트 편집기로 열립니다

✅ **성공 확인:** schema.sql 파일 내용이 보이면 성공입니다!

---

## 🎬 4단계: 첫 번째 스키마 복사하기

### 4-1. 파일 내용 전체 선택
1. schema.sql 파일이 열려있는 편집기에서
2. **Ctrl + A** 키를 누릅니다
   - 이렇게 하면 파일 내용 전체가 선택됩니다
   - 선택된 텍스트는 파란색으로 표시됩니다

### 4-2. 내용 복사하기
1. 선택된 텍스트가 파란색으로 표시되어 있는지 확인
2. **Ctrl + C** 키를 누릅니다
   - 이렇게 하면 내용이 클립보드에 복사됩니다
3. ✅ 복사 완료!

**또는 마우스로:**
1. 파일 내용을 마우스로 드래그해서 전체 선택
2. 마우스 오른쪽 버튼 클릭
3. **"복사"** 또는 **"Copy"** 클릭

---

## 🎬 5단계: Supabase SQL Editor에 붙여넣기

### 5-1. Supabase 브라우저 탭으로 돌아가기
1. 브라우저 탭을 확인합니다
2. Supabase SQL Editor가 열려있는 탭을 찾습니다
3. 그 탭을 클릭합니다

### 5-2. SQL 입력 영역 클릭
1. SQL Editor 화면 가운데에 큰 텍스트 입력 영역이 있습니다
2. 그 영역을 클릭합니다
   - 커서가 깜빡이기 시작하면 준비 완료

### 5-3. 내용 붙여넣기
1. **Ctrl + V** 키를 누릅니다
   - 또는 마우스 오른쪽 버튼 클릭 → **"붙여넣기"** 또는 **"Paste"**
2. ✅ 붙여넣기 완료!
   - SQL 코드가 텍스트 영역에 나타납니다

**확인:**
- SQL 코드가 보여야 합니다
- 여러 줄의 코드가 있어야 합니다
- `CREATE TABLE`, `INSERT INTO` 같은 SQL 명령어가 보여야 합니다

---

## 🎬 6단계: 첫 번째 스키마 실행하기

### 6-1. "Run" 버튼 찾기
1. SQL Editor 화면 상단 또는 하단을 보세요
2. **"Run"** 또는 **"실행"** 버튼을 찾습니다
   - 버튼 색상: 보통 초록색 또는 파란색
   - 아이콘: 재생 아이콘 (▶) 또는 없음
   - 텍스트: **"Run"** 또는 **"실행"**
   - 위치: 보통 오른쪽 상단 또는 하단

### 6-2. "Run" 버튼 클릭
1. **"Run"** 버튼을 클릭합니다
2. ⏳ 잠시 기다립니다 (1-2초)

### 6-3. 결과 확인
1. 화면 오른쪽 또는 아래에 결과가 표시됩니다
2. **성공 메시지 확인:**
   - ✅ **"Success"** 또는 **"성공"**
   - ✅ **"Query executed successfully"**
   - ✅ 또는 초록색 체크 표시 (✓)
   - ✅ 영향받은 행 수 표시 (예: "0 rows affected")

3. **에러가 나면:**
   - ❌ 빨간색 에러 메시지가 나타납니다
   - 에러 메시지를 읽어보세요
   - 보통 SQL 문법 오류일 수 있습니다

✅ **성공 확인:** 성공 메시지가 나타나면 첫 번째 스키마 실행 완료!

---

## 🎬 7단계: 두 번째 스키마 실행하기

### 7-1. 새 쿼리 다시 만들기
1. **"+ New query"** 버튼을 다시 클릭합니다
2. 새로운 빈 쿼리 창이 열립니다

### 7-2. 두 번째 스키마 파일 열기
1. 파일 탐색기로 돌아갑니다
2. `supabase` 폴더 안에서 **"schema_subscriptions.sql"** 파일을 찾습니다
3. **"schema_subscriptions.sql"** 파일을 더블클릭해서 엽니다

### 7-3. 두 번째 스키마 복사하기
1. 파일 내용 전체 선택: **Ctrl + A**
2. 내용 복사: **Ctrl + C**
3. ✅ 복사 완료!

### 7-4. 두 번째 스키마 붙여넣기
1. Supabase SQL Editor로 돌아갑니다
2. 새로 만든 쿼리 창의 텍스트 영역을 클릭합니다
3. **Ctrl + V** 키를 눌러서 붙여넣습니다
4. ✅ 붙여넣기 완료!

### 7-5. 두 번째 스키마 실행하기
1. **"Run"** 버튼을 클릭합니다
2. ⏳ 잠시 기다립니다
3. ✅ 성공 메시지 확인!

## 🚨 문제 해결

### 문제 1: SQL Editor를 찾을 수 없어요

**해결:**
1. 왼쪽 메뉴를 확인하세요
2. "SQL Editor" 또는 "SQL 편집기"를 찾으세요
3. 메뉴가 접혀있을 수 있으니 스크롤해보세요

### 문제 2: "New query" 버튼이 안 보여요

**해결:**
1. SQL Editor 페이지가 제대로 열렸는지 확인
2. 페이지를 새로고침해보세요 (F5 키)
3. 다른 위치에 있을 수 있으니 화면 전체를 둘러보세요

### 문제 3: 파일을 찾을 수 없어요

**해결:**
1. 파일 경로 확인:
   ```
   C:\Users\polor\field-nine-solutions\supabase\schema.sql
   ```
2. 파일 탐색기에서 정확한 경로로 이동하세요
3. 파일 이름이 정확한지 확인하세요

### 문제 4: 복사-붙여넣기가 안 돼요

**해결:**
1. Ctrl+A로 전체 선택했는지 확인
2. Ctrl+C로 복사했는지 확인
3. SQL Editor 텍스트 영역을 클릭했는지 확인
4. Ctrl+V로 붙여넣기 시도

### 문제 5: "Run" 버튼을 찾을 수 없어요

**해결:**
1. 화면 상단과 하단을 모두 확인
2. 오른쪽 상단 또는 하단을 확인
3. 키보드 단축키 사용: **Ctrl + Enter** (일부 에디터에서 작동)

### 문제 6: 에러가 발생해요

**해결:**
1. 에러 메시지를 읽어보세요
2. SQL 코드가 모두 붙여넣어졌는지 확인
3. 파일을 다시 열어서 확인
4. 처음부터 다시 시도
