## ✅ 방법 1: 프로젝트를 프로 플랜 조직으로 이동 (권장)

### 1단계: 프로 플랜 조직 선택

1. **Supabase 대시보드 접속**
   - https://supabase.com/dashboard 접속
   - 로그인

2. **프로 플랜 조직 선택**
   - "kongks5798@gmail.com" (프로 플랜 • 5개 프로젝트) 클릭
   - 또는 프로 플랜이 있는 다른 조직 선택

### 2단계: 새 프로젝트 생성 (프로 플랜 조직에서)

1. **"새로운 프로젝트"** 버튼 클릭
2. **프로젝트 정보 입력:**
   - **이름:** `field-nine-solutions` 또는 `trendstream`
   - **데이터베이스 비밀번호:** 기억할 수 있는 비밀번호 설정
   - **리전:** `ap-northeast-1` (서울) 또는 원하는 리전
3. **"새 프로젝트 생성"** 클릭
4. **프로젝트 생성 완료 대기** (약 2-3분)

### 3단계: 환경 변수 업데이트

#### 3-1. 새 프로젝트 정보 가져오기

1. 새로 생성한 프로젝트 선택
2. 왼쪽 메뉴: **"Settings"** (톱니바퀴 아이콘) 클릭
3. **"API"** 메뉴 클릭
4. 다음 정보를 복사:
   - **Project URL:** `https://xxxxx.supabase.co`
   - **anon public 키:** 긴 문자열

#### 3-2. Vercel 환경 변수 업데이트

1. **Vercel 대시보드 접속**
   - https://vercel.com 접속
   - `field-nine-solutions` 프로젝트 선택

2. **Settings → Environment Variables** 클릭

3. **기존 변수 수정:**
   - `NEXT_PUBLIC_SUPABASE_URL` 찾기
   - **Value**를 새 프로젝트 URL로 변경
   - **"Save"** 클릭

4. **기존 변수 수정:**
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` 찾기
   - **Value**를 새 프로젝트 anon 키로 변경
   - **"Save"** 클릭

### 4단계: 스키마 실행

1. **Supabase SQL Editor 열기**
   - 새 프로젝트에서 왼쪽 메뉴: **"SQL Editor"** 클릭
   - **"New query"** 버튼 클릭

2. **첫 번째 스키마 실행**
   - 프로젝트 폴더에서 `supabase/schema.sql` 파일 열기
   - 내용 전체 복사 (Ctrl+A, Ctrl+C)
   - Supabase SQL Editor에 붙여넣기 (Ctrl+V)
   - **"Run"** 버튼 클릭
   - ✅ 성공 메시지 확인

3. **두 번째 스키마 실행**
   - **"New query"** 버튼 다시 클릭
   - `supabase/schema_subscriptions.sql` 파일 열기
   - 내용 전체 복사
   - 붙여넣기
   - **"Run"** 버튼 클릭
   - ✅ 성공 메시지 확인

### 5단계: 배포 재확인

1. **Vercel 자동 재배포 확인**
   - 환경 변수가 변경되면 자동으로 재배포됩니다
   - 또는 수동으로 재배포:
     - Deployments → 최신 배포 → "Redeploy" 클릭

2. **헬스 체크 확인**
   ```
   https://fieldnine.io/api/health
   ```
   - 데이터베이스가 "healthy"로 표시되면 성공

## ✅ 방법 2: 기존 프로젝트를 프로 플랜 조직으로 이전

**주의:** Supabase는 프로젝트를 직접 이동할 수 없습니다. 
따라서 방법 1(새 프로젝트 생성)을 권장합니다.

만약 기존 데이터를 유지해야 한다면:
1. 데이터베이스 덤프 다운로드
2. 프로 플랜 조직에 새 프로젝트 생성
3. 데이터베이스 덤프 복원

## 💡 팁

1. **프로젝트 이름은 기억하기 쉽게**
   - 예: `field-nine-solutions`, `trendstream`, `fieldnine`

2. **데이터베이스 비밀번호는 안전하게 보관**
   - 나중에 필요할 수 있습니다

3. **환경 변수 업데이트 후 자동 재배포**
   - Vercel이 자동으로 재배포합니다
   - 약 2-3분 소요

4. **기존 무료 플랜 프로젝트는 나중에 삭제 가능**
   - 더 이상 사용하지 않으면 삭제하여 정리

## 🚨 주의사항

1. **기존 데이터가 있다면**
   - 새 프로젝트는 빈 데이터베이스입니다
   - 기존 데이터를 유지해야 한다면 덤프/복원 필요

2. **환경 변수 업데이트 후**
   - 기존 배포는 잠시 오류가 발생할 수 있습니다
   - 재배포가 완료되면 정상 작동합니다

3. **무료 플랜 프로젝트**
   - 나중에 삭제하거나 그대로 두어도 됩니다
   - 무료 플랜은 프로젝트 수 제한이 있습니다
