# /project:autofix - 에러 자동 탐지 + 수정

에러를 탐지하고 자동으로 수정합니다. 에러란 없다.

## 실행 순서

### 1. TypeScript 에러 수정
```bash
npx tsc --noEmit 2>&1
```
- 에러가 있으면 각 에러의 파일과 라인을 읽고 자동 수정
- 수정 후 다시 검증 (최대 3회 반복)

### 2. 런타임 에러 탐지
```bash
npx vitest run 2>&1
```
- 실패 테스트가 있으면 원인 분석 후 수정
- 테스트 코드 문제 → 테스트 수정
- 프로덕션 코드 문제 → 프로덕션 코드 수정
- 수정 후 다시 테스트

### 3. 빌드 에러 수정
```bash
npx next build 2>&1
```
- 빌드 실패 시 에러 분석 후 수정
- RSC 직렬화 에러, import 문제, 타입 에러 등
- 수정 후 다시 빌드

### 4. API 엔드포인트 헬스체크
각 주요 API가 정상 응답하는지 확인:
- `/api/health` — 200 확인
- `/api/ai/chat` — API 키 존재 확인
- `/api/lab/tournaments` — 200 확인

### 5. 보안 체크
- `.env`, `.env.local`이 git에 포함되지 않았는지 확인
- 하드코딩된 API 키가 없는지 확인

## 수정 원칙
- 에러가 발생하면 반드시 수정한다
- 수정 후 반드시 재검증한다
- 같은 에러가 3회 반복되면 다른 접근법을 시도한다
- 모든 수정은 기존 기능을 깨뜨리지 않아야 한다
